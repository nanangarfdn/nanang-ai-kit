Run project health check: lint + typecheck + test.

Usage: /health-check

## Steps

1. **Detect project commands** from:
   - `package.json` scripts (`lint`, `typecheck`/`type-check`/`tsc`, `test`)
   - `Makefile` targets
   - `pyproject.toml` tool configs
   - Fall back to common defaults per detected stack

2. **Run in sequence** (stop on first failure):

   ### Lint
   - Node: `npm run lint` or `npx eslint .` or `npx biome check .`
   - Python: `ruff check .` or `flake8`
   - Go: `golangci-lint run`
   - Rust: `cargo clippy`

   ### Typecheck
   - TypeScript: `npx tsc --noEmit`
   - Python: `mypy .` or `pyright`
   - Go: `go vet ./...`
   - Rust: `cargo check`

   ### Test
   - Node: `npm test` or `npx vitest run` or `npx jest --watchAll=false`
   - Python: `pytest`
   - Go: `go test ./...`
   - Rust: `cargo test`

3. **Report results**:
   ```
   Lint:      ✓ passed  (or ✗ N errors)
   Typecheck: ✓ passed  (or ✗ N errors)
   Test:      ✓ passed  (or ✗ N failed)
   ```

## Rules
- If a check is not applicable (no linter configured), skip it
- Show first 20 lines of errors, not full output
- If `.claude/commands/nng-verify.md` exists (generated by /nng-init-nanang-ai), use those commands instead
